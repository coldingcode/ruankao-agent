# ✏️ 智能评分命令 (Grade Command)

## 功能概述

本命令读取考生在 `index.html` 上的作答数据，调用评分智能体对选择题、问答题、论文题进行**判断对错、打分并提供详细解析**。

## 使用方法

```bash
/grade
```

或指定作答文件：

```bash
/grade --answers output/ruankao_answers_xxx.json
```

## 评分范围

| 部分 | 满分 | 评分方式 |
|-----|------|---------|
| 选择题 | 75 分 | 自动判分（每题 1 分） |
| 问答题 | 75 分 | 要点式评分（选 3 题，每题 25 分） |
| 论文题 | 75 分 | 综合评分 |
| **总分** | **225 分** | 合格线通常为 135 分左右 |

## 前置条件检查

执行评分前，命令会自动检查以下文件是否存在：

```bash
# 必需文件
output/answers.json              # 考生作答
output/exam_manifest.json        # 考试元数据
output/exam_context_pack.json    # 评分上下文包（含答案和评分标准）

# 如缺失则提示
⚠️ 未找到作答文件，请先完成答题并导出 answers.json
⚠️ 未找到考试上下文，请先运行 /generate 生成试卷
```

## 输出文件

1. `output/score_report.json` - 结构化评分数据
2. `output/score_report.html` - 考生友好的评分报告

## 评分报告内容

### JSON 报告结构

```json
{
  "exam_info": {
    "exam_id": "ruankao_20260329_001",
    "candidate": "考生姓名",
    "graded_at": "2026-03-29T14:00:00Z"
  },
  "scores": {
    "part1_mcq": {...},
    "part2_essay": {...},
    "part3_paper": {...}
  },
  "summary": {
    "total_score": 0,
    "total_max_score": 225,
    "percentage": 0,
    "pass_estimate": "通过/未通过",
    "strengths": [],
    "weaknesses": [],
    "recommendations": []

```

### HTML 报告包含

- 总分概览卡片（三科分数）
- 合格/不合格判定
- 各部分得分详情
- 每道题的作答与解析
- 问答题要点评分明细
- 论文题各维度评分
- 学习建议和改进方向

## 相关智能体

- `ruankao-agent:grader-agent` - 评分智能体（实际执行评分）

## 安全说明

评分命令需要访问包含答案的上下文包：
- `exam_context_pack.json` 包含所有答案和评分标准
- 此文件应妥善保管，避免考生访问
- 评分完成后建议将上下文包移至安全位置

## 错误处理

| 错误 | 原因 | 解决方案 |
|-----|------|---------|
| `未找到 answers.json` | 考生未导出作答 | 在 HTML 界面点击"导出作答"按钮 |
| `exam_id 不匹配` | 作答与试卷版本不一致 | 重新生成试卷或重新答题 |
| `context_pack 损坏` | 文件被修改或删除 | 重新运行 /generate 生成试卷 |

## 注意事项

- 评分要公正客观，避免过于严苛或宽松
- 解析要详尽，帮助考生理解错误原因
- 建议要具体可操作
- 对于问答题和论文题，要承认合理答案的多样性
